Это тот уровень, где код перестаёт быть “набором команд” и начинает **принимать решения**, проверять условия, управлять состоянием и корректно работать с ошибками ввода.

Цель урока и материалов в этой папке — не просто перечислить операторы, а **потренировать мышление программиста**:

- как проектировать простую логику так, чтобы её было легко тестировать и расширять.

## Что было в лекции

### 1) Операторы (Operators)
Мы разобрали основные группы операторов Python и научились читать/писать выражения:

- **Арифметические**: `+ - * / ** // %`
- **Сравнения**: `< > <= >= == !=`
- **Присваивания**: `= += -= ...`
- **Логические**: `and / or / not`
- **Принадлежности**: `in / not in`
- **Тождественности**: `is / is not`
- **Битовые**: `& | ^ ~ << >>`

Отдельный акцент — на том, что операторы используются не “в вакууме”, а внутри условий, циклов, проверок входных данных и т.д.

### 2) Ветвление (if)
Конструкция `if` в Python — это **инструкция управления потоком** (control flow statement).  
Она использует выражение, которое вычисляется в `bool` (или приводится к логическому значению через truthiness), и выбирает, какую ветку выполнить.

Именно связка “**операторы + if**” чаще всего и даёт практическую пользу.

### `hw.py`
Домашнее задание из условия:

> Ввести режим вентилятора 0..3 и вывести соответствующее сообщение

…расширено до небольшого консольного приложения **Smart Fan Controller**, чтобы отработать:

## Smart Fan Controller — как использовать

Запуск:

```bash
python lesson2/hw.py
````

Команды в приложении:

* `0..3` — установить режим
* `up` — увеличить режим на 1 (максимум 3)
* `down` — уменьшить режим на 1 (минимум 0)
* `status` — показать текущий режим
* `history` — показать последние изменения
* `stats` — показать статистику (время по режимам, turbo count, энергия)
* `power` — текущая мощность (W) + накопленная энергия (Wh)
* `export` — сохранить статистику в файл рядом с логом запуска
* `help` — подсказка
* `q` / `quit` — выход

### История и логирование

Каждый запуск создаёт отдельный лог-файл (папка `logs/`).
В лог пишутся ключевые события: старт, смена режима, экспорт, выход.

### Статистика и энергия

Статистика считается “по времени”, проведённому в каждом режиме.
Энергия (`Wh`) вычисляется по простой модели мощности:

* режим `0` → 0 W
* режим `1` → … W
* режим `2` → … W
* режим `3` → … W

Это **симуляция**, полезная для тренировки расчётов, времени и аккуратной архитектуры (логику можно расширять и усложнять постепенно).
